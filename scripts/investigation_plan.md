# インポート失敗の根本原因調査

12万件のデータがインポートされたと表示されるにもかかわらず、データベース上のレコードが0件である問題の調査です。

## [ ] 調査項目
- [ ] 標準 `pg` ドライバでの単一レコード挿入・即時検索テスト
- [ ] 挿入時の全エラーメッセージのファイル出力と詳細確認
- [ ] データベース側のテーブル定義（制約、トリガー）の再確認
- [ ] 環境変数（接続文字列）が正しいデータベースを指しているかの最終確認
- [ ] Prisma クライアントのデバッグモード（`DEBUG="prisma*"`）での挙動確認

## 想定される原因
1. **暗黙のロールバック**: ドライバまたは Neon 側で何らかのエラーによりロールバックされている。
2. **接続先の不一致**: インポート先と確認先のデータベース/スキーマが異なっている。
3. **パラメータ型の不一致**: 送信しているデータ型が Postgres 側で静かに拒否されている。
